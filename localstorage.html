Local Storage API - Storing and Retrieving Simple Data, Arrays & Objects

Local Storage

The Web Storage API allows a domain to store data on a local machine using either the localStorage or sessionStorage properties of the Storage object. Data stored using localStorage has no expiration date and persists between sessions, allowing data to be accessed at any future visit to the domain providing persistence and consistency across sessions.

To store data in local storage:

localStorage.setItem('keyName', value);

To read data from local storage:

var myValue = localStorage.getItem('keyName');

To remove a key from local storage:

localStorage.removeItem('keyName');

To remove all keys from local storage:

localStorage.clear();

Arrays & Objects

The Storage API only supports strings for the key/value pairs, so Objects and Arrays should be serialized using JSON.stringify() before writing to storage and deserialized with JSON.parse(). This allows properties and array elements to be modified and accessed as usual before the object/array is committed to storage as a single key/value pair.

Primitive Data Types

Primitive data types are trivial to use with localStorage. Simply assign the value to a key:

localStorage.setItem("browser", "Google Chrome");
localStorage.setItem("version", "80.0.3987.116");
localStorage.setItem("max_storage", 5120);

And read them back the same way:

let currentBrowser = localStorage.getItem("browser");
let currentVersion = localStorage.getItem("version");
let maxStorage = localStorage.getItem("max_storage");
