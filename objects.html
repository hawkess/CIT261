JavaScript - Object Creation, Inheritance, Properties, Methods, Instantiation

Properties

Objects in JavaScript are a way to create a collection of properties that describe an abstract item. Object properties are given names which serve as keys, and values may be attached to those keys upon creation or assigned to the property later. If a key is created with no value assigned to it, its value is considered undefined. Because properties are created as key/value pairs, they are stored as strings and can be written out in JSON. The properties of an object are accessed using what is known as dot notation because it takes the form objectName.objectProperty.

var studentRecord = new Object();
studentRecord.firstName = "John";
studentRecord.lastName = "Smith";
studentRecord.gpa = 3.2;
studentRecord.gradYear = 2024;

Creation/Instantiation

Objects can be created in a few different ways. The example above uses the new keyword to create a new object and then bootstraps new properties by assigning values to them. Objects can also be created with existing properties by assigning an object literal to an object variable.

var usedCar = { make: "Honda", model: "Civic", year: "2005" mileage: "204,281"};

Objects can be created from other objects by using the built-in create() method. This creates a new object with the same properties as the object that was passed as a parameter to the method, and then this object can be modified separately.

let studentTwo = Object.create(studentRecord);

Another option is using a constructor function along with the new keyword to create an instance of an object. This is particularly helpful when a number of objects with the same properties need to be created and initialized with different values. By passing in values as parameters, each instance of the object can be given different starting values for each property. Constructor functions are named after the base object with a capital letter to denote it is a constructor function. The this keyword used in the example refers to the object that the function was called on, in this case the new Employee object.

function Employee(firstName, lastName, position, salary) {
  this.firstName = firstName;
  this.lastName = lastName;
  this.position = position;
  this.salary = salary;
}

var firstEmployee = new Employee("Armando", "Garbanzo", "Marketing Director", 250000);
var secondEmployee = new Employee("John", "Jingleheimerschmidt", "VFX Artist", 80000);

Methods

Functions can be assigned as properties just like other values, allowing you to call a function by using the dot notation to access that property. These are generally used to increase encapsulation by tying functionality to the object rather than having free functions that are called by passing the object.

var myPatient = {
  patientID: 0005101940,
  heightInFeet: 6.2,
  weightInPounds: 210,
  BMI: function() {
    return (this.heightInFeet * 12) / (Math.pow(this.weightInPounds, 2));
  }
}

Inheritance

The functionality of objects can be further extended through inheritance. Inheritance allows objects to take on the properties of the objects they inherit from, while retaining their own separate identity and the ability to define their own properties. Every object in JavaScript inherits from some basic prototype which is what gives JavaScript much of its base functionality, but it is also possible to define a parent for an object to inherit from. This is part of object-oriented programming and allows for relationships between objects to be represented and abstracted.

function Animal(species, color, area) {
  this.species = species;
  this.color = color;
  this.area = area;
}
