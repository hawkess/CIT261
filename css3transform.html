CSS Transitions, Transforms, & Animations

Transitions

CSS provides a way to create transitions that control the progression and speed of change between two different states of an element. The basic syntax to set all the properties for a transition at once is:
transition: [property] [duration] [timing-function] [delay]
However, the transition properties can be set individually and can define multiple transitions in a single line.

transition-property defines the element's CSS style property to be modified. Multiple CSS properties can be listed and other transition properties will apply to these in sequence.

transition-duration defines the length of time the transition between the two states will take. Multiple timings can be listed and will apply in order to properties listed.

transition-timing-function defines how the transition states are interpolated with the following options:

  • ease: speeds up towards the middle, slows down at the end
  • linear: even speed throughout
  • ease-in: start is slow, speeds up towards the end
  • ease-out: start is fast, slows down towards the end
  • ease-in-out: similar to ease, slow in the beginning and end, speeds up towards the middle
  • cubic-bezier: a custom defined timing based on a cubic-Bezier curve with values between 0 and 1
  • steps: define a number of stops along the transition for a defined duration with jump timing
    • jump-start: first jump happens at start
    • jump-end: last jump happens at end
    • jump-none: no jump at start or end
    • jump-both: jump at both start and end

transition-delay defines the duration before the transition begins once the property's value has changed.

Combining all of these into a transition might look something like this:

.transition-div
{
  background-color: red;
  transition: background-color 2s linear 0.5s;
}
transition-div1
{
  background-color: blue
  transition: background-color 2s linear 0.5s
}

Transforms

CSS transforms can rotate, translate, skew, and scale an element to modify its appearance with each transform having an X, Y, and Z component, making it possible to give elements the illusion of a third dimension. Multiple dimensions of the same transform-function can be defined in a single shorthand statement, and multiple functions can be chained together in the same transform property. A full list of all of the available transforms and accompanying syntax can be found here.

An example transform may look something like this:

transform: rotate(180deg) perspective(200px);

Animations

CSS animations make use of keyframes and animation properties to define a gradual series of changes in any number of style properties that can also include transforms and transitions. Though animations and transitions both define how an element changes, transitions react to changes in a property and define how that change should look whereas animations actively change properties. Some pre-defined animations exist in the form of two animation-name options: slide and bounce. Creating keyframe steps and binding the keyframe to an element with a custom animation-name form the foundation of the animation framework. A much more in-depth look at the animation property and its many options is available here.

An example of an animation and the accompanying element it is applied to:

@keyframes example
{
  0% { width: 100px; }
  25% { width: 200px; }
  50% { width: 300px; }
  75% { width: 400px; }
  100% { width: 500px; }
}

div
{
  background-color: red;
  height: 100px;
  width: 0px;
  animation-name: example;
  animation-duration: 2s;
  animation-delay: 0.5s;
}

Triggering Transitions, Transforms and Animations with JavaScript

Triggering a transition, transform, or animation with JavaScript is as simple as adding a class that contains the properties of the transition, transform, or animation to an element.

.box
{
  background-color: red;
  height: 100px;
  width: 100px;
  transition: all 0.5s ease-in;
}

.animated
{
  animation: rotate 2s infinite linear;
}

@keyframes rotate {
  from
  {
    transform: rotateZ(0deg);
  }
  to
  {
    transform: rotateZ(360deg);
  }
}

By adding the animated class in the previous example to an element, the animation defined by rotate would be triggered. Likewise, removing the class would stop the animation.
